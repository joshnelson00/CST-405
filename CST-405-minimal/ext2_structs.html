Goals & Scope

/* Session 1: parse, type-check, no codegen yet */
struct Point { int x; int y; };

p.x = 3;

Field Offset Model

A struct with n integer fields occupies n × 4 bytes on the stack. The field at position i (0-indexed) lives at offset i × 4 from the struct's base address. For a local variable p with base at -8($fp), field y (index 1) is at -8 + 4 = -4($fp).

Stack diagram — struct Point { int x; int y; }
$fp ──►  [ saved $fp ]       +0
         [ saved $ra ]       -4
         [ p.x        ]      -8   ← base of p, field x offset 0
         [ p.y        ]      -12  ← field y offset 4
         [ other locals ]    ...

Your struct type table stores: name → { field_name → byte_offset, total_size }. When allocating a struct variable, add total_size bytes to the current frame offset instead of 4.

Session 1 — Parsing & Type Checking

🎯

Goal: The compiler parses struct definitions and member accesses, builds the type table, and produces meaningful error messages. Code generation is not required this session.

1

Lexer tokens

In scanner.l add three things:

scanner.l additions
"struct"   { return STRUCT; }
"."        { return DOT; }
"&"        { return AMP; }

Declare these in parser.y with %token STRUCT DOT AMP.

2

Grammar & AST nodes

Add two new grammar rules in parser.y: one for struct definitions (a top-level declaration), one for member access as an lvalue expression.

parser.y — new rules (sketch)
/* struct definition */
struct_def:
    STRUCT ID '{' field_list '}' ';'
    { $$ = makeStructDef($2, $4); }

field_list:
    field_list field_decl  { $$ = appendField($1, $2); }
  | field_decl             { $$ = $1; }

field_decl:
    INT ID ';'  { $$ = makeFieldDecl($2); }

/* member access in expr / lvalue */
expr: expr DOT ID  { $$ = makeMemberAccess($1, $3); }

Add to ast.h: NODE_STRUCT_DEF, NODE_FIELD_DECL, NODE_MEMBER_ACCESS, NODE_ADDR_OF. Each struct-def node carries a name (sval) and a linked list of field-decl children. A member-access node carries the base expression and the field name.

3

Struct type table

In symtab.h add a separate struct registry (not the variable table):

symtab.h additions
typedef struct FieldInfo {
    char  name[64];
    int   offset;     /* byte offset from struct base */
} FieldInfo;

typedef struct StructType {
    char       name[64];
    FieldInfo  fields[32];
    int        numFields;
    int        totalSize; /* bytes */
} StructType;

StructType *lookupStruct(const char *name);
void        registerStruct(StructType *st);

Implement these in symtab.c. When registering a struct, assign offsets incrementally: field 0 → 0, field 1 → 4, field 2 → 8, etc., and set totalSize = numFields * 4.

4

Semantic analysis

In semantic.c, handle these new node types:

  • NODE_STRUCT_DEF: call registerStruct() with the field list. Error if name is already defined.
  • Variable declarations of struct type: look up the struct name, store TYPE_STRUCT plus a pointer to the StructType in the symbol table entry.
  • NODE_MEMBER_ACCESS: look up the base variable, verify it has struct type, look up the field name in its StructType. Error if field not found.
⚠

Common mistake: Do not allow arithmetic between struct values (e.g., p + q). Add a type check that emits an error if both operands of a binary op have struct type.

✓ Session 1 Checkpoint

test_struct1.c — semantic errors expected
struct Point { int x; int y; };
struct Point p;
p.x = 10;
p.z = 5;   /* ERROR: no field 'z' in struct Point */
p.x = p;   /* ERROR: type mismatch */
  • Compiler registers Point with offsets x=0, y=4
  • Access to unknown field z produces a clear error with line number
  • No crash on valid programs — error recovery continues after reporting

Session 2 — Code Generation

🎯

Goal: Generate correct MIPS for struct allocation, field reads, field writes, and passing a struct pointer to a function.

1

Stack frame allocation for structs

In tac.c, when you encounter a variable declaration of struct type, emit a TAC_DECL instruction whose size field equals structType->totalSize (not the default 4). In codegen.c, allocateLocal() already uses the size from the symbol table — make sure it reads totalSize for struct variables so the frame pointer is advanced by the right amount.

2

Field read and write

Add two new TAC opcodes in tac.h:

tac.h
TAC_MEMBER_LOAD,   /* result = base.field  → result = *(base + offset) */
TAC_MEMBER_STORE,  /* base.field = src     → *(base + offset) = src   */

In tac.c, NODE_MEMBER_ACCESS generates a TAC_MEMBER_LOAD when used as an rvalue, and a TAC_MEMBER_STORE when used as an lvalue (in an assignment). Store the byte offset as an integer in the instruction's offset field.

In codegen.c, handle these two opcodes:

codegen.c — MEMBER_LOAD
/* TAC_MEMBER_LOAD: result = base + offset */
int baseOff  = getFrameOffset(instr->arg1);   /* e.g. -8 */
int fieldOff = instr->offset;                  /* e.g.  4 */
int totalOff = baseOff - fieldOff;            /* -8 - 4 = -12 */
int destReg  = allocReg(instr->result);
fprintf(out, "\tlw\t%s, %d($fp)\n", regName(destReg), totalOff);
⚠

Watch the sign: getFrameOffset() returns a negative offset from $fp (e.g., -8). Field offset is positive. The final address is baseOffset - fieldOffset.

3

Passing a struct by pointer

Add NODE_ADDR_OF to the AST. In tac.c, emit a TAC_ADDR_OF instruction: result = &var. In codegen.c, implement it using addiu (add immediate unsigned) to compute the address:

codegen.c — TAC_ADDR_OF
fprintf(out, "\taddiu\t%s, $fp, %d\n",
        regName(destReg), getFrameOffset(instr->arg1));

Inside a function receiving a struct pointer (type struct Foo *), member accesses through that parameter work identically to direct access: the base address is the parameter value (already the address), and the offset is added to it.

✓ Session 2 Checkpoint

test_struct2.c
struct Rect { int w; int h; int area; };

void computeArea(struct Rect *r) {
    r.area = r.w * r.h;
}

struct Rect box;
box.w = 6;
box.h = 7;
computeArea(&box);
print(box.area);   /* expected: 42 */
  • Struct fields occupy contiguous stack bytes; field write followed by read produces the same value
  • Address-of produces the correct stack address (verify in MIPS debugger)
  • Function modifies the struct through the pointer; change is visible at call site

Challenge Extensions

  • Nested structs: a field whose type is another struct
  • Pass struct by value (copy all fields onto the callee's stack frame)

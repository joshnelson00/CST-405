1.1 The lowering strategy

Every high-level control structure must be "lowered" to flat three-address code using only three primitives. Study the pattern before coding it:

If without else

t0 = [condition]
IF_FALSE t0 GOTO L_end
    [then-block code]
L_end:

One label, one conditional jump, no GOTO.

If-else

t0 = [condition]
IF_FALSE t0 GOTO L_else
    [then-block code]
GOTO L_end
L_else:
    [else-block code]
L_end:

Two labels, one conditional jump, one unconditional jump.

✏ Task 1.2 — Add the NODE_IF case to generateTACStmt() in tac.c

Find the switch(node->type) in generateTACStmt() (look near the NODE_WHILE case for reference) and add:

        case NODE_IF: {
            /* Allocate two labels:
             *   labelElse → jump target when condition is false
             *   labelEnd  → jump target after the entire if-else */
            char* labelElse = newLabel();
            char* labelEnd  = newLabel();

            /* Step 1: evaluate the condition expression */
            char* cond = generateTACExpr(node->data.if_stmt.condition);

            /* Step 2: emit the conditional branch
             * - if there IS an else, jump to labelElse on false
             * - if there is NO else, jump to labelEnd on false   */
            if (node->data.if_stmt.else_stmt) {
                appendTAC(createTAC(TAC_IF_FALSE, cond, labelElse, NULL));
            } else {
                appendTAC(createTAC(TAC_IF_FALSE, cond, labelEnd,  NULL));
            }
            freeTemp(cond);

            /* Step 3: generate code for the then-branch */
            generateTACStmt(node->data.if_stmt.then_stmt);

            /* Step 4: if else exists — jump over it, emit else label, generate else */
            if (node->data.if_stmt.else_stmt) {
                appendTAC(createTAC(TAC_GOTO,  labelEnd,  NULL, NULL));
                appendTAC(createTAC(TAC_LABEL, NULL,      NULL, labelElse));
                generateTACStmt(node->data.if_stmt.else_stmt);
            }

            /* Step 5: emit the end label — all paths converge here */
            appendTAC(createTAC(TAC_LABEL, NULL, NULL, labelEnd));
            break;
        }

2.1 How constant folding feeds dead-branch elimination

The optimizer makes two passes over the TAC list:

  1. Constant folding: evaluates binary operations where both operands are constants.
    Example: t0 = 5 > 3 → t0 = 1
  2. Dead branch elimination: when IF_FALSE has a constant condition, the branch is either always taken or never taken.

Always false condition

; Input TAC
t0 = 0
IF_FALSE t0 GOTO L_end

; After optimization
t0 = 0
GOTO L_end    ← always jump

Always true condition

; Input TAC
t0 = 1
IF_FALSE t0 GOTO L_end

; After optimization
t0 = 1
; IF_FALSE removed entirely

🐛 Root Cause of the Bug

The register allocator is linear — it processes TAC instructions one by one, left to right, with no knowledge of control flow. When it processes ASSIGN r = 99 it records "r is in $t0". When it then processes PRINT r it finds r in $t0 and emits move $a0, $t0.

But at runtime, if the branch was taken, the assignment never ran. $t0 holds a completely different value. The fix: at every label (merge point), clear the register descriptor table so subsequent reads reload from memory.

✏ Task 3.3 — Implement invalidateAllRegisters() in codegen.c

Add this function near the other register management functions (before generateMIPSFromTAC):

/* Invalidate all register descriptors at a control-flow merge point.
 *
 * WHY: A label can be reached from multiple paths (fall-through or branch).
 * We cannot assume any register holds the value the allocator tracked.
 * Clearing all descriptors forces every subsequent variable read to emit
 * a fresh lw from its stack slot — which is always correct because every
 * variable assignment also does an sw to memory.
 *
 * This is the key correctness fix for if-statements and all other
 * control flow that creates merge points.
 */
static void invalidateAllRegisters() {
    fprintf(output, "    # Invalidate register state at merge point\n");
    for (int i = 0; i < NUM_TEMP_REGS; i++) {
        regAlloc.regs[i].inUse   = 0;
        regAlloc.regs[i].isDirty = 0;
        regAlloc.regs[i].varName[0] = '\0';
    }
}

✏ Task 3.4 — Also add spillAllRegisters() for pre-branch cleanup

Before a branch we should ensure any dirty variable registers are flushed to memory. Add this function immediately before invalidateAllRegisters:

/* Spill all dirty registers to memory before a branch instruction.
 * Ensures that both the taken and fall-through paths find current
 * values in memory when they reach a label. */
static void spillAllRegisters() {
    fprintf(output, "    # Spill all live registers before branch\n");
    for (int i = 0; i < NUM_TEMP_REGS; i++) {
        if (regAlloc.regs[i].inUse && regAlloc.regs[i].isDirty) {
            /* Only spill named variables, not TAC temporaries (t0, t1…) */
            const char* vn = regAlloc.regs[i].varName;
            if (!(vn[0] == 't' && vn[1] >= '0' && vn[1] <= '9')) {
                spillReg(i);
            }
        }
    }
}

✏ Task 3.5 — Call the new functions at the right points

Find the TAC_LABEL, TAC_IF_FALSE, TAC_IF_TRUE, and TAC_GOTO cases in the big switch inside generateMIPSFromTAC() and add the calls:

TAC_LABEL — call invalidateAllRegisters() after emitting the label:

case TAC_LABEL: {
    fprintf(output, "%s:    # Label / merge point\n", curr->result);
    /* Invalidate all registers — code may have arrived here via
     * a branch from a different path than fall-through. */
    invalidateAllRegisters();
    break;
}

TAC_IF_FALSE — call spillAllRegisters() after the branch:

case TAC_IF_FALSE: {
    /* ... load condReg, emit beqz ... */
    fprintf(output, "    beqz $t%d, %s\n", condReg, curr->arg2);
    freeReg(condReg);
    spillAllRegisters();   /* flush state before crossing the branch */
    break;
}

TAC_GOTO — call spillAllRegisters() before the jump:

case TAC_GOTO: {
    spillAllRegisters();   /* flush before unconditional jump */
    fprintf(output, "    j %s\n", curr->arg1);
    break;
}

✅ Why This Fixes the Bug

After invalidateAllRegisters() runs at the label, findVarReg("r") returns -1 (no register holds r anymore). The code generator then falls back to the memory-load path: lw $t0, offset($sp). Since every ASSIGN instruction also does a sw, memory always has the correct latest value of r — whether or not the branch was taken.

🔬 Reflection Questions

  1. Why does invalidateAllRegisters() call spillReg for dirty registers but simply clear the descriptor for clean ones? What could go wrong if you called spillReg for everything?
  2. If you added a proper liveness analysis pass (tracking which variables are live at each label), could you avoid the blanket invalidation? What information would you need?
  3. The current code always stores a variable to memory on assignment. How would your approach change if you introduced lazy stores (only write to memory when the register is needed for something else)?
  4. Trace the MIPS for a nested if-else with three levels of nesting. Count how many extra lw instructions the invalidation strategy introduces compared to a perfect register allocator. Is the trade-off worth it for an educational compiler?

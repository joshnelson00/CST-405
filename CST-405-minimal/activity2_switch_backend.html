Core Concepts

Semantic checks needed for switch

Three rules to enforce

  • Duplicate case values — two case 5: labels in the same switch is an error. Keep a seen-values array and check each new case against it.
  • Duplicate default — a switch can have at most one default: clause.
  • Break context — break is only valid inside a switch, for, or while. Use a depth counter: increment on entry, decrement on exit, error if break is found at depth 0.
  • Missing default (warning only) — a switch without a default silently ignores unmatched values. Warn the programmer.

TAC pattern for switch

A switch is lowered to a linear-scan dispatch table followed by the case bodies in order. The dispatch table compares the controlling expression against each case constant and jumps directly to the matching body. Case bodies are laid out sequentially so that fall-through (no break) is achieved simply by not emitting a jump at the end of a body.

# ─── Example: switch(x) { case 1: P(10); break; case 2: P(20); default: P(0); }

# Evaluate controlling expression (stored in a named variable to survive labels)
DECL __sw0
__sw0 = x

# ─── Dispatch table ───
t0 = __sw0 == 1
IF_FALSE t0 GOTO L_test2
GOTO L_body0 # case 1 matched
L_test2:
t0 = __sw0 == 2
IF_FALSE t0 GOTO L_done
GOTO L_body1 # case 2 matched
L_done:
GOTO L_default # no case matched → go to default

# ─── Case bodies ───
L_body0: # body for case 1
  PRINT 10
  GOTO L_end # ← emitted by NODE_BREAK
L_body1: # body for case 2 (no break → falls through to default)
  PRINT 20
L_default: # default body
  PRINT 0
L_end:
⚠

Why a named variable for the controlling expression? TAC temporaries (t0, t1…) live only in registers. Every TAC_LABEL in the code generator calls invalidateAllRegisters(), so a temporary would be lost after the first label in the dispatch table. By storing the result in a declared variable (__sw0), the code generator always reloads it from its stack slot — correctly, on any path.

The break label stack

A break statement must jump to the end of its enclosing switch or loop. Because switches can be nested inside loops and vice versa, you need a stack of "break targets":

Concept — break label stack operation
/* Entering a switch */
push(L_switch_end);
    generate_case_bodies();   // any break here GETOs L_switch_end
pop();

/* Entering a for loop */
push(L_for_end);
    generate_body();          // any break here GOTOs L_for_end
pop();

/* break inside a switch inside a for loop */
// Stack: [L_for_end, L_switch_end]
// peek() returns L_switch_end → break exits the switch, not the loop ✓

Task 2 — Break Label Stack tac.c

Open tac.c. Add the three stack helper functions near the top of the file, just before the forward declarations section.

tac.c — break label stack
/* ── Break Label Stack ─────────────────────────────────────
 * Tracks the "end" label of each enclosing switch/for/while.
 * NODE_BREAK emits GOTO to peekBreakLabel().
 * Push on entry to each construct, pop on exit.
 */
#define MAX_BREAK_DEPTH 64
static char* breakLabelStack[MAX_BREAK_DEPTH];
static int   breakLabelTop = 0;

static void pushBreakLabel(char* label) {
    if (breakLabelTop < MAX_BREAK_DEPTH)
        breakLabelStack[breakLabelTop++] = label;
    else
        fprintf(stderr, "TAC Error: break stack overflow\n");
}

static char* peekBreakLabel() {
    return (breakLabelTop > 0) ? breakLabelStack[breakLabelTop - 1] : NULL;
}

static void popBreakLabel() {
    if (breakLabelTop > 0) breakLabelTop--;
}
📌

Important: The label strings stored in this stack are heap-allocated by newLabel(). Do not free them before emitting the end label — the TAC instruction that references them keeps pointers to the same string via strdup() inside createTAC().

Task 3 — Switch TAC Generation tac.c

Inside generateTACStmt(), add a case NODE_SWITCH: handler before the existing default: at the bottom. Work through these steps:

1

Evaluate the controlling expression into a named variable

tac.c — step 1
/* Declare a hidden variable to hold the switch value across labels */
char switchVar[32];
sprintf(switchVar, "__sw%d", tacList.labelCount);
appendTAC(createTAC(TAC_DECL, NULL, NULL, switchVar));

char* exprVal = generateTACExpr(node->data.switch_stmt.expr);
appendTAC(createTAC(TAC_ASSIGN, exprVal, NULL, switchVar));
freeTemp(exprVal);

Why not a TAC temp? TAC temps disappear across labels because the code generator calls invalidateAllRegisters() at every TAC_LABEL. A declared variable lives in memory and is always reloaded correctly.

2

Pre-allocate one body label per case

tac.c — step 2
/* Count cases and allocate arrays */
int caseCount = 0;
ASTNode* c = node->data.switch_stmt.cases;
while (c) { caseCount++; c = c->data.case_clause.next; }

char** bodyLabels = (char**)malloc(sizeof(char*) * caseCount);
ASTNode** caseArr  = (ASTNode**)malloc(sizeof(ASTNode*) * caseCount);
int   defaultIdx  = -1;

c = node->data.switch_stmt.cases;
for (int i = 0; i < caseCount; i++) {
    bodyLabels[i] = newLabel();
    caseArr[i]    = c;
    if (c->data.case_clause.isDefault) defaultIdx = i;
    c = c->data.case_clause.next;
}

char* labelDispatchDone = newLabel();
char* labelEnd          = newLabel();
3

Emit the dispatch table

For each non-default case: emit a comparison, a conditional branch to the "next test" label (or labelDispatchDone for the last case), and a GOTO to the matching body label.

tac.c — step 3 (skeleton)
/* Collect indices of non-default cases */
int* nonDef    = (int*)malloc(sizeof(int) * caseCount);
int  nonDefCnt = 0;
for (int i = 0; i < caseCount; i++)
    if (!caseArr[i]->data.case_clause.isDefault)
        nonDef[nonDefCnt++] = i;

for (int ni = 0; ni < nonDefCnt; ni++) {
    int     i   = nonDef[ni];
    ASTNode* cas = caseArr[i];

    /* TODO: build the case constant string */
    char caseConst[32];
    sprintf(caseConst, "%d", cas->data.case_clause.value);

    /* TODO: emit TAC_EQ: cmpTemp = switchVar == caseConst */

    /* Determine fail label: next test or dispatch done */
    char* failLabel = (ni < nonDefCnt - 1) ? newLabel() : labelDispatchDone;

    /* TODO: emit TAC_IF_FALSE: if NOT equal, jump to failLabel */

    /* TODO: emit TAC_GOTO to bodyLabels[i] (match found) */

    /* Emit fail label (used as next case's test entry) */
    if (ni < nonDefCnt - 1)
        appendTAC(createTAC(TAC_LABEL, NULL, NULL, failLabel));
}

/* No case matched → go to default or end */
appendTAC(createTAC(TAC_LABEL, NULL, NULL, labelDispatchDone));
if (defaultIdx >= 0)
    appendTAC(createTAC(TAC_GOTO, bodyLabels[defaultIdx], NULL, NULL));
else
    appendTAC(createTAC(TAC_GOTO, labelEnd, NULL, NULL));

Your job

Fill in the three TODO sections. Use allocTemp() / freeTemp() for the comparison temporary. Look at the existing NODE_IF handler for reference on emitting TAC_EQ + TAC_IF_FALSE.

4

Emit case bodies

Push labelEnd as the break target, then emit each case body in original order. Fall-through is automatic: if a body ends without a NODE_BREAK, execution simply falls to the next label.

tac.c — step 4
pushBreakLabel(labelEnd);
for (int i = 0; i < caseCount; i++) {
    appendTAC(createTAC(TAC_LABEL, NULL, NULL, bodyLabels[i]));
    if (caseArr[i]->data.case_clause.body)
        generateTACStmt(caseArr[i]->data.case_clause.body);
}
popBreakLabel();

appendTAC(createTAC(TAC_LABEL, NULL, NULL, labelEnd));

free(bodyLabels);
free(caseArr);
free(nonDef);
break;

Verify fall-through

Create a small test with a fall-through case and look at the generated assembly:

fall_through_test.c
int main() {
    int x;
    x = 4;
    switch (x) {
        case 4:              // no break → falls through
        case 5: print(99); break;
        default: print(0);
    }
    return 0;
}

What to look for in the MIPS output

  • The label for case 4's body should appear immediately before the label for case 5's body with no jump between them — that is fall-through.
  • The dispatch table should have two beqz instructions (one per non-default case) before the dispatch-done label.
  • Only case 5's body should contain a j L_end (the break). case 4's body should be empty.

Common bugs

Debugging checklist

  • Segfault during TAC generation → check that you free() all three malloc'd arrays (bodyLabels, caseArr, nonDef) after the end label is emitted.
  • Wrong value printed → the switch variable (__sw0) is being read before the controlling expression is stored. Make sure TAC_ASSIGN appears before any comparison instructions.
  • All cases print the default body → the dispatch table is jumping to the wrong body label. Double-check that bodyLabels[i] corresponds to caseArr[i].
  • break inside loop exits the whole function → you forgot to push the loop's labelEnd onto the break stack in NODE_FOR / NODE_WHILE.
  • "TAC Error: break without enclosing context" → the semantic phase should have caught this; check that breakDepth is incremented in NODE_SWITCH before checking case bodies.

Challenge

Optimize the Dispatch: Dense Jump Table Detection

Your current implementation always uses a linear scan — it compares the switch variable against every case constant in order. Real compilers switch strategies based on the density of case values. A dense set of cases (e.g., 1, 2, 3, 4, 5) can be dispatched in O(1) using a jump table; a sparse set benefits from a binary search or linear scan.

Your challenge — choose one:

  • Option A — Jump table detection: Before emitting the dispatch table, calculate the density ratio: (number of non-default cases) / (max_value - min_value + 1). If the ratio is ≥ 0.5, print a message like "Note: switch at line N is dense enough for a jump table (density 0.80)". You do not need to generate the actual jump table — just detect and report it.
  • Option B — Warn on implicit fall-through: During TAC generation, after generating each case body, check if the body's last statement is not a NODE_BREAK or NODE_RETURN. If so, print a warning: "Warning: case N at line M falls through to the next case — add break; if intentional."
  • Option C — continue statement: Add a continue statement to the language that jumps to the top of the enclosing for or while loop (skipping the rest of the loop body). This requires a second label stack alongside the break stack. continue inside a switch (with no enclosing loop) should be a semantic error.

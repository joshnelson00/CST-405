================================================================================
IMPLEMENTATION DETAILS
================================================================================

1. PROPER FUNCTION SCOPING
   ------------------------
   Purpose: Allow parameters and local variables to have the same name by
            implementing proper scope management.
   
   Files Modified:
   - parser.y: Added prepareFunctionScope() calls in mid-rule actions for
               all 9 function grammar rules (INT, FLOAT, VOID functions with
               and without parameters, and array-returning functions)
   - symtab.c: Added prepareFunctionScope() to create function entry and
               activate scope before parsing body
   - symtab.c: Modified addFunction() to update existing entries with AST
   - symtab.h: Added prepareFunctionScope() declaration
   - tac.c: Added enterFunction/exitFunction calls in NODE_FUNC TAC generation
   
   Result: Parameters and local variables can now share names without conflicts.
           Each function has its own isolated scope.

2. FUNCTION PARAMETER COUNT VALIDATION
   ------------------------------------
   Purpose: Detect mismatches between function declaration and function calls
            at compile time.
   
   Files Modified:
   - symtab.c: Modified addFunction() to count and store param_count from AST
   - symtab.c: Added validateFunctionCall() to check argument counts
   - symtab.h: Added validateFunctionCall() declaration
   - ast.c: Modified createFuncCall() to count arguments and call
            validateFunctionCall()
   
   Result: Semantic errors are now reported when functions are called with
           incorrect number of arguments, with clear error messages showing
           expected vs. actual argument counts.

3. COMPILE-TIME ARRAY BOUNDS WARNINGS
   -----------------------------------
   Purpose: Warn about constant array indices that are out of bounds during
            compilation.
   
   Files Modified:
   - parser.y: Added bounds checking in array access rule (ID '[' expr ']')
   - parser.y: Added bounds checking in array assignment rule
   
   Checks Performed:
   - Negative indices (index < 0)
   - Indices beyond array size (index >= size)
   - Only for constant (NODE_NUM) indices
   
   Result: Compile-time warnings are displayed for constant out-of-bounds
           indices, helping catch bugs before runtime.

4. RUNTIME ARRAY BOUNDS CHECKING
   ------------------------------
   Purpose: Prevent undefined behavior from out-of-bounds array access at
            runtime.
   
   Files Modified:
   - tac.h: Added TAC_BOUNDS_CHECK opcode
   - tac.c: Emit TAC_BOUNDS_CHECK before ARRAY_READ operations
   - tac.c: Emit TAC_BOUNDS_CHECK before ARRAY_WRITE operations
   - tac.c: Added handling in printTAC functions
   - optimizer.c: Preserve BOUNDS_CHECK in optimized TAC
   - optimizer.c: Generate MIPS bounds checking code (main loop)
   - optimizer.c: Generate MIPS bounds checking code (function loop)
   - mips.h: Added MIPS_BLTZ and MIPS_BGE opcodes
   - mips.c: Added handlers for new branch instructions
   - optimizer.c: Added bounds_error handler with error message
   
   Runtime Checks Generated:
   - bltz $t8, bounds_error  (check if index < 0)
   - bge $t8, $t9, bounds_error (check if index >= size)
   
   Result: Runtime bounds checking prevents crashes and provides clear error
           messages when array bounds are violated during execution.

5. RUNTIME DIVIDE-BY-ZERO CHECKING
   --------------------------------
   Purpose: Prevent division by zero at runtime in generated MIPS code.
   
   Files Modified:
   - tac.h: Added TAC_DIV_CHECK opcode
   - tac.c: Emit TAC_DIV_CHECK before TAC_DIVIDE operations
   - tac.c: Added handling in printTAC functions
   - optimizer.c: Preserve DIV_CHECK in optimized TAC
   - optimizer.c: Generate MIPS divide-by-zero checking code (main loop)
   - optimizer.c: Generate MIPS divide-by-zero checking code (function loop)
   - mips.h: Added MIPS_BEQZ opcode
   - mips.c: Added handler for beqz instruction
   - optimizer.c: Added div_zero_error handler with error message
   
   Runtime Checks Generated:
   - beqz $t8, div_zero_error (check if divisor == 0)
   
   Result: Runtime divide-by-zero checking prevents crashes and provides
           clear error messages when division by zero is attempted.

4. MIPS Generation

   - Descriptive comments in generated assembly

5. Testing
   - Comprehensive test files for each feature
   - Tests cover both positive and negative cases
   - Verifies entire compilation pipeline (parse → TAC → MIPS)

================================================================================
PERFORMANCE IMPACT
================================================================================

Minimal impact on compilation time:
- Symbol table operations remain O(1) with hash-based lookup
- TAC generation adds ~2 instructions per array access/division
- MIPS generation adds ~3 instructions per runtime check

Runtime performance impact:
- Each array access: 2 branch checks (~2-6 cycles if not taken)
- Each division: 1 branch check (~1-3 cycles if not taken)
- Zero overhead when checks pass (normal execution path)
- Prevents undefined behavior that could cause crashes
